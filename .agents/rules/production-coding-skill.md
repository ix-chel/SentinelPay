name: production-coding
description: Write maintainable, scalable, production-ready code

When to use:
- Writing backend services
- Writing frontend components
- Implementing APIs

Rules:

1. Follow clean architecture principles
2. Use modular structure
3. Avoid duplicated logic
4. Apply dependency injection when appropriate
5. Use descriptive naming conventions

- Functions must be small and single-purpose
- Handle errors properly
- Validate inputs
- Avoid hardcoded values
- Separate configuration

Output requirements:

- File structure
- Code implementation
- Explanation of design decisions
