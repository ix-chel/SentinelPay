name: architecture-design
description: Design scalable software architecture before implementation

When to use:
- Starting a new project
- Adding major features
- Designing backend systems
- Designing API services

Instructions:

1. Identify system requirements
   - Functional requirements
   - Non-functional requirements

2. Define system architecture
   - Monolith or Microservices
   - Layered architecture
   - Event-driven architecture if needed

3. Define core components
   - API layer
   - Business logic layer
   - Data access layer
   - External services

4. Define data model
   - Entities
   - Relationships
   - Indexing strategy

5. Define scalability considerations
   - Horizontal scaling
   - Caching strategy
   - Rate limiting

Output format:

Architecture Overview
System Components
Data Flow Diagram
Technology Stack
Scaling Strategy
